题目

题目链接：182. 查找重复的电子邮箱

编写一个 SQL 查询，查找 Person 表中所有重复的电子邮箱。

示例：

id Email
1 a@b.com
2 c@d.com
3 a@b.com

根据以上输入，你的查询应返回以下结果：

Email
a@b.com

说明：所有电子邮箱都是小写字母。

第一版答案

错误原因：where构建过滤条件时列名必须是原有table中的列明，如这里的Id和Email，不能是count或者是别名，所以where和count不能同时使用，count的值作为判断条件时使用having关键词

第二版答案

错误原因：在count函数接受一个列的名字Email作为参数的时候只会计算Email列的行数，只有一个值。以题目中的例子分析，第一行a@b.com对应的count值为3。而后面的二三行没有对应的count，即为null，这里having判断第一行符合条件后面两行不符合，恰巧正确。假若以下方表格为例这里将输出“a@b.com”，是错误的（正确答案为空）。

id Email
1 a@b.com
2 c@d.com
3 d@b.com

正确答案

解析：对于这道题需要先使用group by对Email进行分组，将Email值相同的分到一组。然后在每个分组中使用count(Email)分别计算每个分组的行数，最后通过这个count(Email)的返回值对每个Email分组进行过滤。

其他正确答案

注意事项：新建列的别名紧跟函数后，如这里的的EmailNumber。SQL嵌套创建的新table必须有一个自己的名称。
